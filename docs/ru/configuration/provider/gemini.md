# Конфигурация Gemini API

Используйте параметр `gemini-api-key` для настройки ключей Gemini API. Каждая запись принимает необязательные значения `base-url`, `headers` и `proxy-url`; заголовки прикрепляются только к запросам, отправляемым на переопределенный эндпоинт Gemini, и никогда не пересылаются на прокси-серверы. Устаревший эндпоинт `generative-language-api-key` предоставляет зеркальное отображение только ключей для обратной совместимости — запись через этот эндпоинт обновляет список Gemini, но отбрасывает любые переопределения для конкретных ключей, а устаревшее поле больше не сохраняется в `config.yaml`.
